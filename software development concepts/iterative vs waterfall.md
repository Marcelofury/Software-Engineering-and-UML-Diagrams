# Iterative Development vs Waterfall
## Iterative Development
- Build system in small cycles (iterations)
- Each cycle includes:
- Design → Build → Test → Feedback
## Waterfall Model
- Linear process (one phase at a time)
- No going back easily
### Why Iterative is Better:
- Early feedback from users
- Easier to fix errors
- Flexible to changes
- Faster delivery of usable features
- Lower risk
